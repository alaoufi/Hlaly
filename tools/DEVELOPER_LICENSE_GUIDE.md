# حماية التطبيقات بمفتاح تفعيل (Ed25519) — دليل المطوّر

نظام تفعيل **يعمل بلا إنترنت**، مربوط بالجهاز، وآمن: التطبيق يحمل **المفتاح العامّ فقط**
ويتحقّق محلياً، والبذرة السرّية تبقى في مولّد الأكواد (keygen) عندك. الكود الواحد لا يعمل
إلا على الجهاز الذي صُدر له.

---

## ١) كيف يعمل (باختصار)
- لكل جهاز **رقم جهاز** ثابت (١٦ حرفاً Base32) يعرضه التطبيق في شاشة التفعيل.
- أنت تولّد **كوداً** موقّعاً بمفتاحك السرّي لذلك الجهاز ولمدّة معيّنة.
- التطبيق يتحقّق من التوقيع **بالمفتاح العامّ فقط** (لا يحتاج إنترنت ولا يحوي السرّ).
- المدّة بالأيام: **0 = ترخيص دائم**، أو رقم أيام (١–٦٥٥٣٥).

**النصّ المُوقَّع:** `PREFIX|رقم_الجهاز|المدّة` — مثال: `UNIV1|TESTDEVICE234567|30`.

**الحزمة (٦٦ بايت):** بايتان للمدّة (Big‑Endian) + ٦٤ بايت توقيع Ed25519، ثم تُرمَّز Base32
بأبجدية مخصّصة: `ABCDEFGHJKLMNPQRSTUVWXYZ23456789`.

---

## ٢) خطوات الإعداد لمرّة واحدة
```bash
npm i tweetnacl
node keygen.mjs new
```
يطبع:
- **المفتاح العامّ (Base64):** ضعه في التطبيق (license.js → `PUB_B64`).
- **البذرة السرّية (64 hex):** احتفظ بها في المولّد فقط — **لا تضعها في التطبيق إطلاقاً**.

> اختر بادئة (`PREFIX`) خاصّة بكل تطبيق (مثل `MYAPP1`) لتفصل تراخيصه، أو بادئة موحّدة
> (`UNIV1`) إن أردت مفتاحاً واحداً يفعّل عدّة تطبيقات.

---

## ٣) توليد كود لجهاز
اطلب من المستخدم **رقم الجهاز** الظاهر في شاشة التفعيل، ثم:
```bash
# ترخيص دائم:
node keygen.mjs code --seed <البذرة_hex> --device DEVICEID16CHARS --days 0 --prefix UNIV1

# ترخيص 30 يوماً:
node keygen.mjs code --seed <البذرة_hex> --device DEVICEID16CHARS --days 30 --prefix UNIV1
```
انسخ «الكود» وأعطه للمستخدم؛ يُدخله في التطبيق فيعمل فوراً بلا إنترنت.

---

## ٤) جانب التطبيق (المتحقّق)
التطبيق يحتاج `tweetnacl` (nacl.min.js) و`license.js` بالثوابت التالية:
```js
var PUB_B64 = '<المفتاح_العامّ_Base64>';   // العامّ فقط — آمن للنشر
var PREFIX  = 'UNIV1';                      // نفس بادئة المولّد
var B32     = 'ABCDEFGHJKLMNPQRSTUVWXYZ23456789';
```
المنطق (مبسّط):
```js
function verify(code, deviceId) {
  var pkt = base32Decode(norm(code));            // norm: أحرف كبيرة + احذف ما ليس [A-Z0-9]
  if (pkt.length !== 66) return null;
  var dur = (pkt[0] << 8) | pkt[1];              // المدّة بالأيام (0 = دائم)
  var sig = pkt.slice(2);                         // 64 بايت
  var msg = enc(PREFIX + '|' + norm(deviceId) + '|' + dur);
  return nacl.sign.detached.verify(msg, sig, b64ToBytes(PUB_B64)) ? dur : null;
}
```
- **رقم الجهاز** يشتقّه التطبيق ويعرضه: يكفي أن يكون ثابتاً لكل تثبيت (١٦ حرف Base32).
  (في حلالي: ١٠ بايت عشوائية مخزّنة → Base32. ويمكن اشتقاقه من عتاد الجهاز:
  `Base32(SHA256("ملح:" + hardwareId)[0..10])` ليثبت عبر إعادة التثبيت.)
- **المدّة تُفحص عند كل فتح** مع حارس ساعة: `effNow = max(now, lastSeen)` كي لا يمدّد
  إرجاعُ ساعة الجهاز المدّة. التفعيل المخزَّن لا يُعاد التحقّق من كوده، فلا يتأثّر المفعَّلون
  عند تغيير المفاتيح لاحقاً.

---

## ٥) دعم عدّة تطبيقات
- **مفاتيح منفصلة:** لكل تطبيق زوج مفاتيح وبادئة مستقلّة (عزل كامل).
- **مفتاح موحّد:** نفس `PUB_B64` والبادئة في كل التطبيقات → كود واحد يفعّلها جميعاً على
  نفس الجهاز. (اجعل التطبيق يقبل أكثر من مجموعة مفاتيح للتوافق الرجعيّ عند التبديل.)

---

## ٦) ملاحظات أمنية
- انشر **المفتاح العامّ** بأمان؛ لا يمكن توليد أكواد منه.
- **احمِ البذرة السرّية** (offline، نسخة احتياطية مشفّرة). من يملكها يولّد الأكواد.
- الأكواد مربوطة بالجهاز؛ كود جهازٍ لا يعمل على آخر.
- لا يمنع النظام النسخ الاحتيالي المتقدّم (تعديل الـAPK)، لكنه يكفي للحماية التجارية العملية.
- ضمانة المالك: أضف استرجاعاً بالبذرة (يدخل المالك البذرة فيُفعَّل دائماً) كي لا يُحبَس أبداً.
