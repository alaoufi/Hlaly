# التفعيل الجاهز — ٣ خطوات فقط (بلا أي اجتهاد)

الحزمة كاملة في هذا المجلد: `guard.js` + `nacl.min.js` (+ المولّد `keygen.mjs` منفصل عندك).

## مرّة واحدة: أنشئ مفتاحك
```bash
npm i tweetnacl
node keygen.mjs new
```
انسخ سطر **PUB** (المفتاح العامّ). احتفظ بـ **SEED** (البذرة السرّية) عندك سرّاً — لا تضعه في التطبيق.

## الخطوات (داخل تطبيقك)
**١) انسخ ملفين** إلى مجلد تطبيقك **بجانب `index.html`**:
`guard.js` و `nacl.min.js`

**٢) افتح `guard.js` وغيّر سطرين فقط** (أعلى الملف):
```js
var PUBLIC_KEY = 'ضع_PUB_هنا';
var APP_NAME   = 'اسم تطبيقك';
```

**٣) أضف سطرين في `<head>` من `index.html`** — **قبل** سكربتات تطبيقك:
```html
<script src="nacl.min.js"></script>
<script src="guard.js"></script>
```

## انتهى ✅
- عند فتح التطبيق **غير المفعَّل** تظهر **شاشة تفعيل تلقائياً** تغطّي التطبيق: تعرض «رقم الجهاز» + حقل الرمز.
- المستخدم يرسل لك رقم الجهاز، فتولّد له كوداً:
  ```bash
  node keygen.mjs code --seed <SEED> --device <رقم_الجهاز> --days 0     # دائم
  ```
- يُدخل الكود → يُفعَّل ويعمل بلا إنترنت. الكود مربوط بجهازه فقط.

> `PREFIX` في `guard.js` و`keygen.mjs` يجب أن يتطابقا (الافتراضي `UNIV1`).
> لتعطيل الحماية أثناء التطوير: اترك `PUBLIC_KEY` كما هو (REPLACE_...) فيمرّ التطبيق بلا شاشة.
