# مراحي 🐪🐑🐐🐄

تطبيق **Android** مستقل وخفيف لإدارة الحلال (إبل، غنم، ماعز، بقر)، يعمل **بدون إنترنت** ويخزّن كل البيانات **محلياً على الجهاز**، مع نسخ احتياطي ومشاركة.

> مشروع منفصل تماماً عن بقية المستودع. كل ملفاته داخل مجلد `marahi/`.

## الفكرة
متابعة كل بهيمة بشكل فردي من **الولادة حتى البيع أو النفوق**، مع التركيز على:

**الترقيم → النسب → الحمل → الولادة → التطعيم → العلاج → التحريم**

بدون محاسبة أو مبيعات أو إدارة معقدة.

## المزايا
- **سجل البهيمة**: النوع، رقم المراح، المعرّف (رقم / وسم / شريحة / اسم — مهم للإبل)، الجنس، تاريخ الميلاد، اللون، الحالة، النسب (الأم والأب)، الملاحظات.
- **النسب التلقائي**: عند فتح سجل الأم تظهر مواليدها (أنتجت: ١٢٥، ١٤٤، ...).
- **التلقيح والحمل**: تسجيل التلقيح + حساب **مدة الحمل وتاريخ الولادة المتوقع تلقائياً** حسب النوع.
- **الولادات**: تسجيل المولود وربطه بالأم تلقائياً (وإضافته كبهيمة جديدة).
- **التطعيمات**: تعريف أنواع التطعيمات مرة واحدة + إعطاء تطعيم مع **حساب انتهاء التحريم تلقائياً**.
- **العلاجات**: نوع/اسم العلاج + مدة التحريم + الإجراء + **حساب انتهاء التحريم**.
- **التنبيهات**: ولادة متوقعة خلال ٧ أيام، انتهاء مدة التحريم، مواعيد تطعيم قادمة.
- **البحث**: برقم / وسم / شريحة / اسم البهيمة.
- **النسخ الاحتياطي**: تصدير **JSON** و **Excel (CSV)**، استعادة، ومشاركة عبر واتساب/البريد.
- **عربي بالكامل** مع واجهة من اليمين لليسار (RTL).

## التقنية
- **Kotlin + Jetpack Compose (Material 3)**
- **Room** للتخزين المحلي (SQLite) — أوفلاين 100%
- **kotlinx.serialization** للنسخ الاحتياطي
- لا أذونات إنترنت، ولا خوادم — كل شيء على الجهاز.

## بناء الـ APK

### الطريقة الأسهل: GitHub Actions
ادفع الكود إلى GitHub، ثم من تبويب **Actions** شغّل workflow باسم **Build Marahi APK**.
بعد انتهائه، نزّل الملف من قسم **Artifacts** باسم `marahi-debug-apk`.

### محلياً (يتطلب Android SDK / Android Studio)
```bash
cd marahi
./gradlew assembleDebug
# الناتج: app/build/outputs/apk/debug/app-debug.apk
```

## التثبيت على الجوال
انقل `app-debug.apk` إلى الجوال وثبّته (فعّل «تثبيت من مصادر غير معروفة»).
